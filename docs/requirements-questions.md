# 需求澄清清单

本文档记录工程化改造前需要确认的问题。未确认的问题不会被当作最终需求实现。

## 1. 用户与权限

- 待确认：系统的用户类型有哪些？
  - 例如：普通用户、管理员、知识库管理员、开发/运维人员。
- 待确认：是否需要企业统一登录？
  - 例如：LDAP、SSO、OAuth、飞书/钉钉/企业微信登录。
- 待确认：是否需要权限隔离？
  - 例如：不同用户只能访问自己的 workspace，不同部门访问不同知识库。

## 2. 核心业务能力

- 待确认：当前 Agent 的正式业务名称是什么？
- 待确认：首期必须保留的能力有哪些？
  - 本地知识库问答。
  - 文件上传与管理。
  - 文档翻译。
  - SPI log 解析。
  - 代码/脚本辅助 Agent。
  - 联网搜索。
- 待确认：哪些能力属于 MVP，哪些可以后续迭代？

## 3. 知识库与 RAG

- 待确认：知识库是否按用户、项目、车型、客户或部门隔离？
- 待确认：知识库文档是否需要版本管理？
- 待确认：是否需要人工审核入库？
- 待确认：是否需要展示答案引用来源、页码、片段和置信度？
- 待确认：当前 Chroma 是否可以短期保留，还是需要立即迁移到 Qdrant/Milvus？

## 4. 文件与任务

- 待确认：上传文件的最大大小是多少？
- 待确认：需要支持哪些文件格式？
- 待确认：翻译任务是否需要异步执行和进度展示？
- 待确认：SPI log 解析结果是否需要持久化、下载和历史查询？
- 待确认：文件是否需要长期保存，还是任务完成后可清理？

## 5. 并发与可用性

- 待确认：预期同时在线用户数是多少？
- 待确认：预期并发对话数是多少？
- 待确认：单次问答可接受的最大响应时间是多少？
- 待确认：文档解析/翻译任务可接受的最大处理时间是多少？
- 待确认：目标可用性是多少？
  - 例如：工作时间可用、99%、99.9%。

## 6. 部署环境

- 待确认：最终部署在什么环境？
  - 公司内网服务器。
  - Windows 服务器。
  - Linux 服务器。
  - Docker Compose。
  - Kubernetes。
- 待确认：是否允许访问外部模型 API？
- 待确认：是否有内网代理、证书或网络限制？
- 待确认：是否需要离线部署或私有化模型？

## 7. 安全与合规

- 待确认：业务文档是否包含敏感信息？
- 待确认：是否需要操作审计日志？
- 待确认：是否需要敏感词、脱敏或数据防泄漏策略？
- 待确认：API Key 应由用户输入、管理员配置，还是平台统一托管？

## 8. 验收标准

- 待确认：首期工程化改造完成的验收标准是什么？
  - 可以启动前后端。
  - 可以登录。
  - 可以对话。
  - 可以上传文件。
  - 可以执行知识库问答。
  - 有基础单元测试。
  - 有 Docker Compose 一键启动。
